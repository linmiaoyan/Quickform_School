# QuickForm 模块：已认证教师流程与可见内容

本文档说明**已认证教师**（`is_certified = True`）在系统中的整体流程与各处能看到、能做的内容。

---

## 一、入口与导航

已认证教师登录后，与普通用户共用同一套导航，但会多出/解锁以下能力：

| 位置 | 已认证教师看到/能做的 |
|------|------------------------|
| **顶部导航** | 首页、官方文档、开源教程、精品案例、**项目交流**、**数据任务**、创建任务、个人设置、退出 |
| **数据任务（仪表盘）** | 我的任务列表 + **「创建新任务」** + **「一键生成新任务（内测中）」** 按钮 |
| **个人中心（个人设置）** | 显示“您已完成教师身份认证”，可配置学校/手机/邮箱、AI 模型与 API 等 |

未认证用户：仪表盘只有「创建新任务」，没有「一键生成新任务」；创建任务时不能选“公开到项目交流”。

---

## 二、核心流程概览

```
登录
  → 数据任务（仪表盘）
       ├─ 创建新任务（表单创建，可选 仅自己 / 组织 / 公开到项目交流）
       └─ 一键生成新任务（内测）※ 仅认证教师可见
  → 进入某个任务（任务详情）
       ├─ 查看简介、数据接口地址、POST 提示词、网页/HTML
       ├─ 导出数据、智能分析、编辑任务
       ├─ 共享管理：分配给组织、分享给用户、项目公开范围（可设「公开到项目交流」）
       └─ 若为 AI 生成页：可「AI 继续修改」HTML（剩余次数内）
  → 项目交流（社区）
       └─ 浏览他人公开项目、点赞、发帖/回复
```

---

## 三、已认证教师专属/强化的能力

### 1. 一键生成新任务（内测）

- **入口**：数据任务页右侧按钮「一键生成新任务（内测中）」。
- **流程**：填写任务标题 + 页面需求描述，可勾选若干固定说明（如接口用法），提交后系统用 AI（硅基流动等）生成单页 HTML，并自动创建任务、上传该 HTML。
- **权限**：仅管理员或已认证教师可进入；未认证用户点击会提示“请先完成教师认证”。
- **API**：优先用个人中心配置的 API；未配置时使用部署方配置的 `CHAT_SERVER_API_TOKEN`（.env）。
- **审核**：认证教师生成的 HTML **自动通过审核**（`html_approved = 1`），无需管理员再审。

### 2. 公开到项目交流

- **创建任务时**：分享范围可选择「公开（出现在项目交流，所有人可浏览）」；未认证用户该选项为灰色并提示需教师认证。
- **任务详情页**：在「共享管理」里可把「项目公开范围」设为「公开到项目交流」并保存；公开后需管理员审核通过才会在「项目交流」展示。

### 3. HTML 审核与「AI 继续修改」

- **上传的 HTML**：认证教师上传的页面**自动通过审核**，不进入待审核队列。
- **编辑任务页**：若任务带 AI 生成的 HTML，可看到「AI 继续修改」及剩余次数；认证教师可正常使用，不受“未认证需先审核”限制。

### 4. 任务数量与邮箱

- **任务上限**：与普通用户一样受 `task_limit` 限制（默认 3），管理员可单独调高或设为无限制。
- **第二个及以后任务**：若账号未绑定真实邮箱（或仅为占位邮箱），会先提示绑定/验证邮箱；认证状态不豁免该校验。

## 四、各页面简要说明（已认证教师视角）

| 页面 | 说明 |
|------|------|
| **首页** | 产品介绍、快速开始、关于；可进入数据任务或注册/登录。 |
| **数据任务（dashboard）** | 我的任务列表（自己+组织+他人共享），每个任务可复制接口地址、查看详情、导出、智能分析；顶部有「创建新任务」和「一键生成新任务（内测中）」。 |
| **创建任务（create_task）** | 表单创建：标题、简介、分享范围（私有/组织/公开）。认证教师可选「公开」；可上传 HTML 或后续在编辑页上传。 |
| **一键生成新任务（oneclick_create_task）** | 仅认证教师可访问。填标题+需求，勾选说明项，提交后自动创建任务并生成 HTML，跳转任务详情。 |
| **任务详情（task_detail）** | 标题、简介、数据接口 URL、POST 提示词（中文引号包裹网址）、网页/HTML 展示、导出/分析/编辑、共享管理（组织、指定用户、公开范围）。公开任务可点赞。 |
| **编辑任务（edit_task）** | 改标题/简介、上传/替换 HTML、组织与分享范围；若为 AI 生成页则显示「AI 继续修改」及剩余次数。 |
| **个人中心（profile）** | 显示已认证状态；可改学校、手机、邮箱，配置 AI 模型与 API；教师认证入口（认证后本页显示“已完成教师身份认证”）。 |
| **项目交流（community）** | 浏览公开项目列表、进入他人任务详情、点赞、发帖与回复。 |

## 五、和未认证用户的差异小结

| 能力 | 未认证用户 | 已认证教师 |
|------|------------|------------|
| 创建任务（表单） | ✅ | ✅ |
| 一键生成新任务 | ❌ 不显示入口 | ✅ 显示并可使用 |
| 分享范围选「公开到项目交流」 | ❌ 灰色不可选 | ✅ 可选 |
| 上传 HTML 后的审核 | 需管理员审核通过才展示 | 自动通过，无需审核 |
| 任务详情里设「公开到项目交流」 | 可设但需管理员审核 | 可设，同样需管理员审核通过才上项目交流 |
| 个人中心 | 可申请教师认证 | 显示已认证，无限制使用上述能力（在 task_limit 内） |

## 六、数据与接口（与是否认证无关）

- 每个任务对应唯一**提交接口**：`POST {host}/api/{task_id}`，用于接收 JSON 数据。
- 任务详情页提供**精简版提示词**（网址用中文引号「」包裹），便于复制给大模型或他人使用。
- 查询数据：`GET .../api/{task_id}`（最近 3 条）、`GET .../api/{task_id}/all`（全部），用于统计、导出与智能分析。

以上为当前逻辑下的已认证教师流程与可见内容总结；若路由或权限有改动，以代码为准。
